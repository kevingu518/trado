# 功能規格文檔

## 功能概述

Trado React MVP 包含兩個核心功能模組：
1. 登入系統
2. 交易紀錄功能

## 1. 登入系統

### 1.1 使用者登入

**功能描述**：使用者透過電子信箱和密碼登入系統

**輸入欄位**：
- 電子信箱（必填，需符合 email 格式）
- 密碼（必填）
- 記住我（選填，checkbox）

**驗證規則**：
- 電子信箱：必填、需符合 email 格式
- 密碼：必填

**行為**：
1. 使用者輸入帳密後點擊「登入」按鈕
2. 系統驗證表單欄位
3. 呼叫登入 API (`POST /auth/login`)
4. 成功後：
   - 儲存 access_token 到 sessionStorage
   - 更新 Redux 認證狀態
   - 顯示成功訊息
   - 導航至 `/trades` 頁面
5. 失敗後：
   - 顯示錯誤訊息
   - 保持登入頁面

**相關檔案**：
- `src/pages/Login/index.jsx`
- `src/api/api_user.js`
- `src/store/authSlice.js`

### 1.2 使用者註冊

**功能描述**：新使用者註冊帳號

**輸入欄位**：
- 電子信箱（必填）
- 密碼（必填）
- 確認密碼（必填）

**驗證規則**：
- 電子信箱：必填、需符合 email 格式
- 密碼：必填、需符合密碼強度要求
- 確認密碼：必填、需與密碼一致

**行為**：
1. 使用者填寫註冊表單
2. 系統驗證表單欄位
3. 呼叫註冊 API (`POST /auth/register`)
4. 成功後：
   - 顯示成功訊息
   - 導航至登入頁面
5. 失敗後：
   - 顯示錯誤訊息

**相關檔案**：
- `src/pages/Register/index.jsx`
- `src/api/api_user.js`

### 1.3 忘記密碼

**功能描述**：使用者忘記密碼時重設密碼

**輸入欄位**：
- 電子信箱（必填）

**行為**：
1. 使用者輸入電子信箱
2. 系統發送重設密碼連結到該信箱
3. 顯示成功訊息

**相關檔案**：
- `src/pages/ForgotPassword/index.jsx`
- `src/api/api_user.js`

### 1.4 路由保護

**功能描述**：保護需要認證的頁面，未登入使用者自動導向登入頁

**實作方式**：
- 使用 `ProtectedRoute` 組件包裹需要保護的路由
- 檢查 Redux 中的 `isAuthenticated` 狀態
- 未認證時導向 `/auth/login`

**相關檔案**：
- `src/components/ProtectedRoute.jsx`
- `src/App.jsx`

## 2. 交易紀錄功能

### 2.1 交易記錄列表

**功能描述**：顯示所有交易記錄的列表

**顯示欄位**：
- 編號（自動計算）
- 方向（多/空，以顏色標示）
- 股號（4 位數字）
- 開倉日
- 清倉日（未清倉顯示 "-"）
- 結果（盈虧金額，以顏色標示）
- 紀律（通過/失敗/未處理，以圖示標示）
- 狀態（持倉中/已完成，以標籤顯示）
- 操作（查看、新增倉位）

**功能**：
- 分頁顯示（預設每頁 10 筆）
- 可展開查看詳細資訊（倉位記錄、交易檢討）
- 點擊行可展開/收合詳細資訊

**相關檔案**：
- `src/pages/Transactions/index.jsx`

### 2.2 開單（建立 Position）

**功能描述**：建立一筆新的交易記錄

**輸入欄位**：
- 股號（必填，4 位數字）
- 多空（必填，LONG/SHORT）
- 策略（必填，下拉選單）
- 開倉日（必填，日期選擇器）
- 清倉日（選填，日期選擇器）
- 備註（選填，文字區域，最多 200 字）

**驗證規則**：
- 股號：必填、必須為 4 位數字
- 多空：必填
- 策略：必填
- 開倉日：必填

**行為**：
1. 點擊「開單」按鈕開啟彈窗
2. 填寫表單欄位
3. 點擊「新增」按鈕
4. 系統驗證表單
5. 成功後：
   - 建立新的 Position 記錄（初始 fills 為空陣列）
   - 將記錄添加到列表最上方
   - 關閉彈窗
   - 顯示成功訊息
6. 失敗後：
   - 顯示驗證錯誤訊息

**資料結構**：
```javascript
{
  key: string,              // 唯一識別碼
  stockCode: string,        // 股號
  direction: 'LONG' | 'SHORT',
  strategy: string,
  openDate: string,         // YYYY-MM-DD
  closeDate: string | null,
  notes: string | null,
  fills: [],                // 初始為空
  review: {                 // 初始檢討資料
    content: '',
    errorCategory: '',
    selfRating: 0,
    emotion: ''
  },
  status: 'open',           // 初始狀態為持倉中
  discipline: 'pending',    // 初始為未處理
  result: null              // 初始為 null
}
```

**相關檔案**：
- `src/pages/Transactions/components/AddPositionModal.jsx`

### 2.3 建倉位（新增 Fill）

**功能描述**：為已存在的交易記錄添加倉位變動

**輸入欄位**：
- 日期（必填，日期選擇器）
- 動作（必填，買入/賣出）
- 價格（必填，數字，小數點後 2 位）
- 數量（必填，整數，大於 0）
- 停損價（選填，數字，小數點後 2 位）
- 備註（選填，文字）

**驗證規則**：
- 日期：必填
- 動作：必填
- 價格：必填、必須大於 0
- 數量：必填、必須大於 0
- 停損價：選填、必須大於 0

**行為**：
1. 在交易記錄列表中點擊「新增倉位」按鈕
2. 開啟建倉位彈窗（顯示當前交易記錄資訊）
3. 填寫倉位資訊
4. 點擊「新增」按鈕
5. 系統驗證表單
6. 成功後：
   - 將 Fill 記錄添加到對應 Position 的 fills 陣列
   - 關閉彈窗
   - 顯示成功訊息
7. 失敗後：
   - 顯示驗證錯誤訊息

**資料結構**：
```javascript
{
  key: string,              // 唯一識別碼
  date: string,            // YYYY-MM-DD
  action: 'buy' | 'sell',
  price: number,
  quantity: number,
  stopLoss: number | null,
  notes: string | null
}
```

**相關檔案**：
- `src/pages/Transactions/components/AddFillModal.jsx`

### 2.4 查看交易詳情

**功能描述**：在抽屜中顯示交易記錄的完整資訊

**顯示內容**：

1. **基本資訊**：
   - 股號
   - 方向
   - 開倉日
   - 清倉日
   - 狀態

2. **持倉統計**：
   - 總數量
   - 平均價格
   - 總價值
   - 策略
   - 備註

3. **盈虧分析**：
   - 盈虧金額
   - 盈虧比例
   - 手續費
   - 淨損益
   - 總資產比例

4. **倉位記錄**：
   - 可編輯的表格
   - 顯示所有 fills
   - 支援新增、編輯、刪除倉位記錄

5. **交易檢討**：
   - 檢討內容（文字區域）
   - 錯誤分類（下拉選單）
   - 當時情緒（下拉選單，帶顏色標示）
   - 是否遵守紀律（開關）
   - 自我評分（1-5 星評分）

**行為**：
1. 在交易記錄列表中點擊「查看」按鈕
2. 從右側開啟抽屜
3. 顯示完整的交易資訊
4. 可編輯倉位記錄和交易檢討
5. 點擊「保存檢討」按鈕保存檢討內容

**相關檔案**：
- `src/pages/Transactions/components/TransactionDrawer.jsx`

### 2.5 過濾與搜尋

**功能描述**：過濾交易記錄列表

**過濾條件**：
1. **日期範圍**：使用 RangePicker 選擇開始和結束日期
2. **股票號碼**：下拉選單，顯示所有已使用的股號
3. **策略**：下拉選單，顯示所有已使用的策略
4. **多空方向**：下拉選單（多/空）
5. **快速過濾器**：下拉選單
   - 全部
   - 僅顯示持倉中
   - 僅顯示已清倉
   - 僅顯示虧損單
   - 僅顯示盈利單

**行為**：
1. 選擇過濾條件
2. 系統即時過濾列表
3. 分頁重置為第一頁

**相關檔案**：
- `src/pages/Transactions/index.jsx` (getFilteredData 函數)

### 2.6 分頁功能

**功能描述**：分頁顯示交易記錄

**功能**：
- 每頁顯示筆數可調整（預設 10 筆）
- 顯示總筆數和當前範圍
- 支援跳轉到指定頁面

**相關檔案**：
- `src/pages/Transactions/index.jsx` (pagination 狀態)

## 3. 交易檢討功能

### 3.1 檢討內容

**功能描述**：記錄交易的心得與檢討

**欄位**：
- 檢討內容（必填，文字區域，最多 1000 字）
- 錯誤分類（必填，下拉選單）
- 當時情緒（必填，下拉選單）
- 是否遵守紀律（必填，開關）
- 自我評分（必填，1-5 星）

**錯誤分類選項**：
- 技術分析錯誤
- 基本面分析錯誤
- 進出場時機錯誤
- 風險控制不當
- 情緒影響判斷
- 策略執行偏差
- 市場環境誤判
- 其他

**情緒選項**：
- 自信（綠色）
- 冷靜（藍色）
- 焦慮（橙色）
- 貪婪（橙紅色）
- 恐懼（紅色）
- 沮喪（紫色）
- 興奮（粉紅色）
- 平靜（灰色）

**行為**：
1. 在交易詳情抽屜中填寫檢討內容
2. 點擊「保存檢討」按鈕
3. 系統驗證必填欄位
4. 成功後顯示成功訊息
5. 更新交易記錄的 review 欄位

**相關檔案**：
- `src/pages/Transactions/components/TransactionDrawer.jsx`
